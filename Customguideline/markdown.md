# CUSTOM DEVELOPMENT GUIDELINES

## MARKDOWN FILES (.md)

### 1. Structure & Formatting
- Use consistent heading hierarchy (# → ## → ### → ####)
- Start documents with H1 title
- Leave blank lines before/after headings
- Use ATX-style headers (# Header) over Setext-style (underlined)
- Limit line length to 80-100 characters for readability

### 2. Content Organization
- Use descriptive, SEO-friendly headings
- Include table of contents for long documents
- Group related content under logical sections
- Use consistent bullet point style (-, *, or +)
- Number ordered lists consistently (1. 2. 3.)

### 3. Code & Links
- Use fenced code blocks with language specification:
```javascript
const example = "Use language tags";
```
- Use relative links for internal documentation
- Validate all external links periodically
- Use descriptive link text (avoid "click here")

### 4. Best Practices
- Use emphasis sparingly (bold for important, italic for emphasis)
- Include alt text for all images: ![Description](image.png)
- Use tables for structured data with proper alignment
- End files with single newline
- Use consistent date formats (YYYY-MM-DD recommended)
